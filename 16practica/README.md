- хранить изображения медиаобъектов

# Media Library System (Личная медиатека)

## Название проекта
13practicaa — Информационная система личной медиатеки

---

## Описание проекта

Проект представляет собой информационную систему для хранения и просмотра личной медиатеки пользователя, включающей:

- фильмы
- сериалы
- книги
- музыку
- видеоигры

Система позволяет просматривать медиаконтент в визуальном формате, отслеживать статусы и рейтинги, а также хранить изображения в бинарном формате (BLOB).

---

## Описание выполненной работы

В рамках практической работы была разработана клиентская часть информационной системы.

Реализовано:

### 1. Страница каталога (CatalogPage.xaml)
- отображение данных из базы данных
- вывод медиаобъектов в ListView
- нестандартный ItemsPanel (WrapPanel)
- формирование сетки карточек

---

### 2. Карточки объектов

Каждая карточка содержит:

- изображение (из BLOB-поля VARBINARY(MAX))
- название медиаконтента
- тип (фильм, сериал, книга, игра, музыка)
- год выпуска
- статус просмотра
- средний рейтинг

При отсутствии изображения используется заглушка.

---

### 3. Работа с изображениями (BLOB)

- изображения хранятся в базе данных в формате VARBINARY(MAX)
- выполняется преобразование бинарных данных в изображение в WPF
- обеспечена корректная визуализация изображений в интерфейсе

---

### 4. Навигация

Реализована навигация:

- после успешной авторизации/регистрации пользователь перенаправляется на страницу каталога (CatalogPage)

---

### 5. Стилизация интерфейса

Используются стили и цветовая палитра, определённые в App.xaml (из практической работы №14), включая:

- оформление карточек
- единый визуальный стиль приложения
- адаптацию элементов интерфейса

---
